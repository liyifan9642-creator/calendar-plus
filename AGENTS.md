# AGENTS.md — VoiceCal 语音日历

## 项目概述

VoiceCal 是一款 AI 驱动的语音日历工具，用户通过自然语言（语音/文字）创建、查询、修改、删除日程事件。系统通过 LLM 智能理解用户意图，自动提取时间、地点等信息。

### 核心场景
- 驾车时语音创建/查询日程
- 运动中语音修改安排
- 会议间隙快速添加待办
- 老年用户说话即可管理日程

---

## 技术栈

| 层级 | 技术 |
|------|------|
| 后端 | Java 17 + Spring Boot 3.2.5 |
| 前端 | React 18 + TypeScript |
| 构建 | Maven（多模块）+ npm |
| 数据库 | H2（开发）/ PostgreSQL（生产） |
| AI | Google Speech ASR + LLM |

### 模块结构

```
voice-calendar/
├── voicecal-app      — 应用入口和配置
├── voicecal-core     — 核心业务逻辑
├── voicecal-nlu      — 自然语言理解（NLU）
├── voicecal-service  — 服务层
├── voicecal-ui       — 前端 UI 模块
├── voicecal-voice    — 语音处理模块
└── frontend/         — React 前端应用
```

### 常用命令

```bash
# 后端构建
mvn clean install              # 构建所有模块
mvn spring-boot:run            # 启动后端（在 voicecal-app 目录）
mvn test                       # 运行测试

# 前端开发
cd frontend
npm install                    # 安装依赖
npm start                      # 启动开发服务器
npm run build                  # 构建生产版本
```

### 代码规范

- 使用 Lombok 减少样板代码
- 使用 MapStruct 进行对象映射
- REST API 遵循 RESTful 设计
- 前端使用 TypeScript 严格模式

## 任务管理

1. **先规划**：将计划写入 `tasks/todo.md`，使用可勾选清单。
2. **验证计划**：实现前先 check-in。
3. **跟踪进度**：每完成一项即标记。
4. **解释变更**：每步提供高层总结。
5. **记录结果**：在 `tasks/todo.md` 末尾添加 review 部分。
6. **捕捉教训**：纠正后更新 `tasks/lessons.md`。

---

## 核心原则

- **简洁优先**：每次变更尽量简单，只影响最小代码。
- **绝不偷懒**：找到根因，不用临时修复，坚持资深开发者标准。
- **最小影响**：只修改必要部分，避免引入新 bug。
